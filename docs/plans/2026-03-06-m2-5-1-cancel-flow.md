# M2.5.1 Cancel Flow Baseline Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Complete `M2.5.1` by making the OpenAI runtime track active streamed runs and expose a real cancel path that future timeout and frontend work can call.

**Architecture:** Keep scope limited to the backend runtime contract that already exists in `services/agent_trigger.py` and `infra/runtime/openai.py`. Use the OpenAI Agents SDK streaming result handle as the cancellation primitive, register it by `request_id`, clean it up in `finally`, and keep the public `AgentRuntime.cancel(run_id)` interface unchanged so `M2.5.2` and `M2.5.3` can build on it without churn.

**Tech Stack:** Python 3.11+, pytest, OpenAI Agents SDK, async generators
